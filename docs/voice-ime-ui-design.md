# ChiVoice 语音输入法 · UI 设计规格

本文档配套 `docs/spec.md` 和 `/Users/mutse/.claude/plans/app-app-imperative-honey.md` 的架构方案，定义 Doubao 风格语音输入法在 chivoice 上的 UI 线框、状态机、运动与视觉规格。

---

## 0. 设计基线（沿用现有水墨风）

| 维度 | 现状（Flutter） | IME / Extension 中如何复刻 |
|---|---|---|
| 底色 | `kPaper #F8F2E6` 纸张米色 + 顶底渐变 | Kotlin `@color/paper`，省略山形剪影（性能与跨进程难度） |
| 主色 | `AppSkin.primaryValue` 6 套（默认 `bamboo #48624B`） | 通过 `SettingsBridge` 从 EncryptedSharedPreferences 读出 hex，运行时 `setTint` |
| 次色 | `AppSkin.secondaryValue` | 同上，用作按钮渐变下角 |
| 墨色文字 | `kInk #223025` / `kInkSoft #687465` | 静态资源 |
| 描边 | `kPaperLine #E0D2BC` | 静态资源 |
| 卡片 | 白 82% 透明 + 圆角 24-26dp + 无投影 | `MaterialCardView`，使用 `android:alpha="0.82"` |
| 圆按钮 | 108dp（待机）↔ 120dp（录制中），白色 + 次色渐变 + 主色描边 + 主色 12-24% 阴影 | `MaterialButton` + `ShapeAppearance` |
| 字体 | `Noto Serif SC` / `Songti SC` 衬线（仅标题） | IME 用系统 fallback `serif`，避免在键盘进程中加载 1.5MB 字体子集 |
| 文案语气 | 文言诗意（"轻声落字，开口即写" / "倾听中…"） | IME 保留少量微文案，更克制（屏幕高度有限） |

## 1. Android IME 面板

### 1.1 尺寸与栅格

- 总高度：**320dp**（竖屏；横屏自适应缩到 220dp 并隐藏 Feature 行）
- 安全区：底部留 `windowInsetsBottom`
- 内边距：左右 16dp，上下 12dp
- 文字层级：title 18sp / body 14sp / chip 12sp

### 1.2 默认（Idle）状态

```
┌─────────────────────────────────────────────────┐
│  🌐 中文  ▾                            ⌨︎ 切回  │ ← 顶栏 40dp
├─────────────────────────────────────────────────┤
│                                                 │
│             轻声落字 · 按住下方说话              │ ← 引导文案
│                                                 │
│              · · · · · · · · · ·                │ ← 静态点阵（暗示波形）
│                                                 │
│                  ╭─────╮                        │
│                  │  🎙 │  ← 96dp 圆按钮         │ ← 按住录音
│                  ╰─────╯                        │
│                                                 │
│  [📋 粘贴]    [✨ 指令]    [⌫]      [✓ 完成]   │ ← 工具行 48dp
└─────────────────────────────────────────────────┘
```

### 1.3 Listening 状态（按住麦克风期间）

```
┌─────────────────────────────────────────────────┐
│  🌐 中文  ▾   ⏱ 00:03              ✕ 取消      │
├─────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────┐    │
│  │ 你好我想问一下今天下午                  │    │ ← Partial 预览（半透明）
│  └─────────────────────────────────────────┘    │
│                                                 │
│  ▁▃▅▇▆▄▂▁▂▄▆▇▅▃▁▂▄▆▇▆▄▂▁ ← 30 条柱波形         │
│                                                 │
│                  ╭═════╮                        │
│                  ║ ⬛  ║  ← 录制中（蓝/主色脉冲环）│
│                  ╰═════╯                        │
│                                                 │
│           松开发送  ·  上滑取消                  │ ← 操作提示
└─────────────────────────────────────────────────┘
```

要点：
- 麦克风按钮从 96 → 108dp，主色描边变粗 4→6dp，外圈脉冲环（120dp，10% 透明度，呼吸 2s）
- 波形使用现有 `WaveformWidget` 的 Kotlin 复刻：30 条柱，80ms `ValueAnimator` 平滑
- Partial 文本仅显示最后 1-2 行，超长截断头部（`ellipsize="start"`）
- **上滑取消**：按住期间手指上移超过 60dp，按钮变红，松开时丢弃录音

### 1.4 Preview 状态（松开后）

```
┌─────────────────────────────────────────────────┐
│  🌐 中文  ▾                          ✕ 重录    │
├─────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────┐    │
│  │ 你好，我想问一下今天下午的会议           │    │
│  │ 改到几点了？                             │    │ ← 最终文本（可编辑）
│  └─────────────────────────────────────────┘    │
│                                                 │
│  [✨ 润色] [更正式] [压缩] [翻译] [总结]   →    │ ← 横滑动作条
│                                                 │
│  [📋 复制]   [⌫ 删一字]          [▶ 送入输入框]│ ← 提交栏
└─────────────────────────────────────────────────┘
```

要点：
- 文本框可点击进入快速编辑（弹出原生光标 + 系统键盘候选区，但 IME 保持当前面板）
- **动作条横向滑动**：8 个 AI 动作（去口语/更正式/压缩/分段/提炼待办/翻译/总结/自定义指令），右侧渐隐提示可滑
- 「送入输入框」= `commitText(finalText, 1)` + 关闭面板
- 「复制」= 写入 `ClipboardManager`，Snackbar "已复制"
- **未决问题 1 答案**（执行前确认）：默认采用「先 commit 原始转写、再点 AI 改写」的两步式，因为 AI 误改不可逆；勾选「自动送出」可一键

### 1.5 Instruction-detected 状态

当 partial 或 final 检测到指令前缀（如「翻译成英文 ...」「总结一下 ...」），UI 自动变形：

```
┌─────────────────────────────────────────────────┐
│  🌐 中文  ▾                          ✕ 重录    │
├─────────────────────────────────────────────────┤
│  ╭─ 🪄 翻译模式 → 英文 ──────────────────╮      │ ← 高亮指令 chip
│  │                                       │      │
│  │ 你好世界                              │      │ ← 去掉前缀的 payload
│  ╰───────────────────────────────────────╯      │
│                                                 │
│             ⏳ AI 整理中 · · ·                  │ ← 占位
│                                                 │
│  [取消指令，仅保留原文]        [▶ 应用结果]    │
└─────────────────────────────────────────────────┘
```

要点：
- 指令 chip 用主色背景（不是低饱和），引人注目
- AI 流式返回时，结果在原文下方逐字渐入（30ms / 字符）
- 「取消指令」回退到普通 Preview 状态

### 1.6 Error 状态

红色横幅从顶部下推 32dp，不挤压主面板：

```
┌─────────────────────────────────────────────────┐
│  ⚠ 网络不可用，已切换为本地识别      [关闭]    │ ← 错误条
├─────────────────────────────────────────────────┤
│  ... (Preview 状态保留)                         │
└─────────────────────────────────────────────────┘
```

错误类型：
- 麦克风权限缺失 → Toast「请到设置授权」+ 跳 MainActivity
- 网络失败 → 顶部横幅 + fallback 到 on-device STT
- AI 失败 → 顶部横幅 + fallback 到规则版 `rewriteDraftOffline`

## 2. AI Studio 底部表（Phase 1 主入口）

入口：录音页 `_DraftState` 的「快捷整理」横排按钮上方加一个 `✨ AI 整理` Filled 按钮，点击弹出。

### 2.1 布局（高度占 90% 屏幕，可下拉关闭）

```
╭───────────────────────────────────────────╮
│              ━━━━ (drag handle)             │
│                                            │
│  ✨  AI 整理                          ✕    │ ← 标题栏
│  ─────────────────────────────────────     │
│                                            │
│  原文                          [💡 自动检测]│ ← 段标题 + 提示 chip
│  ┌──────────────────────────────────────┐  │
│  │ 嗯那个我想问一下今天下午的会议……    │  │ ← 只读卡片
│  │ 改到几点了，参会人员有变动吗？      │  │
│  └──────────────────────────────────────┘  │
│                                            │
│  风格                                       │ ← 段标题
│  ┌──────┬──────┬──────┬──────┐             │
│  │去口语│更正式│ 压缩 │ 分段 │             │ ← 动作网格 2×4
│  ├──────┼──────┼──────┼──────┤             │
│  │待办  │ 翻译 │ 总结 │指令…│             │
│  └──────┴──────┴──────┴──────┘             │
│                                            │
│  AI 输出                       [↻ 重生成]   │
│  ┌──────────────────────────────────────┐  │
│  │ 请问今天下午的会议改到几点了？      │  │ ← 流式渐入
│  │ 参会人员有调整吗？                  │  │ ← 输出区
│  └──────────────────────────────────────┘  │
│                                            │
│  [    取消    ]    [    替换原文 ▶    ]    │ ← 提交栏
╰───────────────────────────────────────────╯
```

要点：
- 「💡 自动检测」chip：若原文包含指令前缀，自动高亮对应的动作格子
- 「翻译」按钮二级展开：底部弹出语种选择条（中/英/日/法/西/德）
- 「指令…」打开自定义 prompt 输入框，让用户写「帮我写一封婉拒邮件」之类
- 「重生成」按钮按 temperature 0.5→0.8 重跑一次（让结果更发散）
- 流式输出过程中，「替换原文」按钮显示 loader spinner 不可点击
- 用户撤销：替换后顶部 Snackbar「已应用，6 秒内点击撤销」（沿用现有模式）

### 2.2 离线降级

- 无网/无 key 时，AI 输出区显示「✕ AI 不可用，已使用本地规则整理」+ 灰底文字
- 仍可正常返回内容（调用 `rewriteDraftOffline`）
- 顶部加 chip「离线模式」提示

入口：用户在任意 app（Safari / Notes / WeChat）选中文本 → 系统分享面板 → ChiVoice 图标。

### 3.1 布局（固定高度 480pt）

```
┌─────────────────────────────────────────────┐
│  ✕                用语音改写                │ ← 顶栏 44pt
├─────────────────────────────────────────────┤
│                                              │
│  原文                                        │
│  ┌────────────────────────────────────────┐ │
│  │ The meeting has been rescheduled to    │ │ ← 选中文本卡
│  │ 3pm. Please confirm your attendance.   │ │
│  └────────────────────────────────────────┘ │
│                                              │
│  说一句你想做什么                            │ ← 引导
│  ┌────────────────────────────────────────┐ │
│  │ 例如：翻译成中文 / 改成更礼貌的语气     │ │ ← 指令 hint
│  └────────────────────────────────────────┘ │
│                                              │
│              ╭─────────╮                     │
│              │   🎙    │                     │ ← 96pt 大圆按钮
│              ╰─────────╯                     │ ← 按住说话
│                                              │
│         ▁▃▅▇▆▄▂▁▂▄▆▇▅▃▁  ← 录制时显示       │
│                                              │
│  [ 仅复制原文 ]      [ 录后自动复制结果 ▶ ]  │ ← 底栏 56pt
└─────────────────────────────────────────────┘
```

### 3.2 状态变化

| 状态 | 麦克风按钮 | 文本区域 |
|---|---|---|
| 待机 | 96pt 白底主色边 | 显示提示语 |
| 录音中 | 108pt 主色实心 + 脉冲 | 显示 partial（来自 SFSpeechRecognizer）|
| 处理中 | 显示 spinner | 显示 "AI 整理中…" |
| 完成 | 还原 | 显示最终文本 |

### 3.3 完成行为

iOS 扩展无法直接编辑宿主 app 文本框。完成时：
1. 把 AI 结果写入 `UIPasteboard.general.string`
2. 调用 `extensionContext.completeRequest(returningItems: nil)` 关闭扩展
3. 系统返回宿主 app
4. 显示系统级 Toast「已复制，长按粘贴」（通过 `UNUserNotificationCenter` 短时通知，2 秒消失）

### 3.4 微距设计细节
- 顶栏 `✕` 按钮：直接调 `extensionContext.cancelRequest`
- 录音前从 App Group 读 `aiBaseUrl/aiApiKey/aiModel`，若未配置，显示居中卡片「请先在 ChiVoice app 配置 AI」+ 跳主 app 按钮
- 录音权限：iOS 14+ 扩展可访问麦克风（与 Share Extension 兼容），首次会弹系统权限框

└─────────────────────────────────────────────────┘
```

### 4.2 交互

- 选择服务商 chip → 自动填入对应 baseUrl 和推荐 model：
  - Groq: `https://api.groq.com/openai/v1` + `llama-3.3-70b-versatile`
  - OpenAI: `https://api.openai.com/v1` + `gpt-4o-mini`
  - DeepSeek: `https://api.deepseek.com/v1` + `deepseek-chat`
  - 自定义: 清空，让用户输入
- 「测试连接」：发一个最小 prompt（"ping" 类）验证响应；成功显示 ✓ 绿色 chip，失败显示错误码 + 提示
- 「[显示]」长按显示明文 3 秒；「[粘贴]」读剪贴板
- AI 增强关闭时，下面所有项灰显但可见

### 4.3 安全
- API key 在 Android 用 `EncryptedSharedPreferences` 存
- iOS 用 Keychain + `kSecAttrAccessGroup` for App Group 共享
- Flutter 端用 `flutter_secure_storage`（已经能跨平台），或直接走平台通道
- Hive 中**不**明文存 key（只存 hint，如 mask 后的 `sk-•••abc`）

## 5. 悬浮球（Phase 4 · Android 可选）

### 5.1 静态形态

```
        ╭───╮
        │ 🎙 │   ← 48dp 圆，主色实心，白色麦克风
        ╰───╯
```

- 默认贴右边缘，垂直居中
- 3 秒无交互 → 透明度降到 60%
- 用户拖动 → 不透明，松开吸附最近的左/右边缘，记忆 Y 坐标

### 5.2 展开形态（点击后）

```
                    ╭──────────────────────╮
                    │  按住说话             │
                    │                       │
                    │       ╭─────╮         │  ← 240dp × 280dp 浮窗
                    │       │ 🎙  │         │
                    │       ╰─────╯         │
                    │                       │
                    │  ▁▃▅▇▆▄▂▁           │
                    │                       │
                    │  [✕] [✨][▶ 写入]    │
                    ╰──────────────────────╯
              ╭───╮
              │ ⬛ │  ← 球本体变为收回状态
              ╰───╯
```

- 长按悬浮球 → 直接进入 Listening（跳过点击展开），松开自动定位结果到剪贴板 + 通知
- 「写入」按钮：若当前 app 焦点是输入框且 chivoice 是默认 IME → commitText；否则 → 剪贴板
- 用户上滑浮窗 → 关闭浮窗回到球状态

### 5.3 权限
- `SYSTEM_ALERT_WINDOW`（`Settings.canDrawOverlays`）
- 设置页加引导卡：「开启悬浮球可在任意 app 上方调用语音输入」+ 跳设置开关

## 6. 运动与微交互规格

| 场景 | 参数 | 备注 |
|---|---|---|
| 状态切换（fade + slide） | 240ms · cubic-bezier(0.4, 0, 0.2, 1) | Material standard |
| 麦克风按钮按下 | scale 1.0→1.08，120ms，触觉反馈 `HapticFeedbackConstants.KEYBOARD_TAP` | |

| 错误横幅 | 从顶部下推 240ms · 自动 5s 收起 | |
| 上滑取消 | 手指 > 60dp 时按钮变红，触觉重提示 | |
| Snackbar | bottom 16dp，6s 持续，撤销操作可点 | |

## 7. 多语种切换 UX

│  [● 中文] [英文] [日文] [法文] [西文] [德文]│ ← 横向 chip row
└─────────────────────────────────────────────┘
```

- 长按系统地球键（Android IME 标准操作）→ 调出系统输入法选择器，与上面无关
- 选中后立即生效，影响 STT 的 `languageCode` 和 AI prompt 的目标语种
